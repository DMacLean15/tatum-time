============================================================
  TATUM TIME — PUTTING THE SITE + WRITING DASHBOARD ONLINE
============================================================

This sets up two things:
  1) The live website at a real web address.
  2) A "write a post" dashboard at  YOURSITE/admin  so Dan can
     publish new posts with NO code and NO Claude.

You only do this setup ONCE. After that, Dan just logs in and writes.

You'll need two FREE accounts:
  • GitHub  (github.com)   — stores the site files
  • Netlify (netlify.com)  — hosts the site + handles logins


------------------------------------------------------------
THE FILES (this whole folder)
------------------------------------------------------------
tatum-time/
├─ index.html              ← the website
├─ content/
│   └─ posts.json          ← all blog posts (the CMS edits this)
├─ admin/
│   ├─ index.html          ← the writing dashboard
│   └─ config.yml          ← the form fields Dan fills in
├─ images/uploads/         ← created when he uploads a photo
├─ DEPLOY-CMS.txt          ← this file
└─ HOW TO ADD A POST.txt   ← the no-CMS backup method


------------------------------------------------------------
STEP 1 — PUT THE FILES ON GITHUB
------------------------------------------------------------
1. Sign in at github.com and click "New repository".
2. Name it (e.g. "tatum-time"), keep it Public, click Create.
3. On the new repo page click "uploading an existing file".
4. Drag in EVERYTHING from the tatum-time folder (keep the
   content/ and admin/ folders intact), then "Commit changes".


------------------------------------------------------------
STEP 2 — DEPLOY ON NETLIFY
------------------------------------------------------------
1. Sign in at netlify.com  ->  "Add new site"  ->  "Import an
   existing project"  ->  GitHub  ->  pick your "tatum-time" repo.
2. Leave the build settings blank (it's a plain site) and click
   "Deploy". In ~30 seconds you get a live address like
   https://something-random.netlify.app
3. (Optional) Site settings -> "Change site name" to make it
   nicer, e.g. https://tatumtime.netlify.app
4. Open admin/config.yml and update the two "site_url" lines to
   your real address, then re-upload that file to GitHub.


------------------------------------------------------------
STEP 3 — TURN ON LOGINS (so only Dan can post)
------------------------------------------------------------
In your Netlify site dashboard:
1. Go to:  Integrations / Identity  ->  Enable Identity.
   (If you don't see "Identity", search the dashboard for it.)
2. Under Identity -> Registration, set it to "Invite only".
3. Under Identity -> Services -> Git Gateway -> click Enable.


------------------------------------------------------------
STEP 4 — INVITE DAN
------------------------------------------------------------
1. Identity tab -> "Invite users" -> type Dan's email -> send.
2. Dan gets an email, clicks the link, and sets a password.
3. From then on he logs in at:   YOURSITE/admin
   (for example  https://tatumtime.netlify.app/admin )


------------------------------------------------------------
HOW DAN WRITES A POST (the whole point)
------------------------------------------------------------
1. Go to  YOURSITE/admin  and log in.
2. Click "Blog Posts" -> "All Posts".
3. Click "Add Posts" to start a new one. Fill in:
      • Headline
      • Banner Emoji   (e.g. 🏀)
      • Category       (pick from the menu)
      • Date
      • Game Box-Score Link  (optional — paste an NBA.com box
        score address and a green "See full game stats" button
        appears automatically; leave blank for none)
      • Article        (a normal editor: bold, headings, lists.
        Tip: wrap a stat in ==  ==  to make it a green tag,
        e.g.  ==28.3 PPG== )
4. Drag the newest post to the TOP of the list.
5. Click "Publish". That's it — the live site updates in under
   a minute. No code, no Claude.


------------------------------------------------------------
(OPTIONAL) TRY THE DASHBOARD ON YOUR COMPUTER FIRST
------------------------------------------------------------
Want to test before going live? In the tatum-time folder:
  • Open one terminal and run:   npx decap-server
  • Open another and run:        python -m http.server 8765
  • Visit:  http://localhost:8765/admin/
This uses a local test login so you can play with the fields.


------------------------------------------------------------
GOOD TO KNOW
------------------------------------------------------------
• The website must be ONLINE (Netlify) for the dashboard to work.
  Opening index.html from your desktop won't load the CMS — it
  falls back to a built-in copy of the posts instead.
• Editing posts in the dashboard rewrites content/posts.json on
  GitHub automatically — you never touch it by hand.
• If Netlify ever hides "Identity" for new sites, the alternative
  is Decap CMS's GitHub login. Ask and it can be reconfigured.
